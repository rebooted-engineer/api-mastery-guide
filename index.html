N+1 happens when you fetch a list (1 query) then fetch related data for each item (N queries). 100 posts = 101 database queries. Senior developers catch this in code review.

The quick reference every senior backend developer keeps in their head. Status codes, headers, patterns, and the questions interviewers love to ask.

HTTP Status Codes

CodeNameWhen to use
200OKSuccessful GET, PUT, PATCH — returns data
201CreatedSuccessful POST that created a resource — include Location header
202AcceptedRequest accepted for async processing — return job ID
204No ContentSuccessful DELETE or action with no response body
301Moved PermanentlyResource has a new permanent URL — client should update bookmarks
304Not ModifiedETag matched — client can use cached version
400Bad RequestMalformed request syntax, invalid format (not validation)
401UnauthorizedNo token or invalid token — authentication required
403ForbiddenValid token but insufficient permissions — you know who they are
404Not FoundResource doesn't exist — or you're hiding it exists (security)
409ConflictState conflict — duplicate email, concurrent edit collision
410GoneResource permanently deleted — unlike 404, it existed before
422UnprocessableValidation failed — well-formed request but semantic errors
429Too Many RequestsRate limit exceeded — include Retry-After header
500Server ErrorUnexpected server error — never return implementation details
502Bad GatewayUpstream service returned invalid response
503Service UnavailableServer down for maintenance or overloaded

HTTP Methods Quick Reference

MethodSafe?Idempotent?Use for
GET✓ Yes✓ YesFetch resource — never modify state
POST✗ No✗ NoCreate resource or trigger action — can create duplicates
PUT✗ No✓ YesReplace entire resource — same result if called 10 times
PATCH✗ No~ MaybePartial update — idempotent only if designed carefully
DELETE✗ No✓ YesRemove resource — calling twice has same result
HEAD✓ Yes✓ YesGet headers only — check if resource exists without body
OPTIONS✓ Yes✓ YesCORS preflight — discover allowed methods
